# Java generated-types compile check

The contract Java records are generated by `contracts/codegen/generate.py` into
`contracts/generated/java/`. They use Jackson annotations
(`com.fasterxml.jackson.annotation.*`), so they do not compile with a bare `javac` —
`javac` reports "package com.fasterxml.jackson.annotation does not exist" unless a
`jackson-annotations` jar is on the classpath.

Full Maven/Gradle packaging (artifact `io.simintero.contracts`) — which supplies the
Jackson dependency on the classpath — is deferred to Section C with registry publishing.
For Section A we only verify the generated sources compile against `jackson-annotations`.

## Verify

Requires a `jackson-annotations` jar on the classpath (any 2.x is fine). Locate one that
is already on disk (the Gradle wrapper ships one; `~/.m2`/`~/.gradle` caches usually have
others):

    JACKSON_JAR=$(find ~/.m2 ~/.gradle -name 'jackson-annotations-*.jar' 2>/dev/null | head -1)
    contracts/codegen/.venv/bin/python contracts/codegen/generate.py
    javac -cp "$JACKSON_JAR" -d /tmp/contracts-java-check $(find contracts/generated/java -name '*.java')

Expected: compiles with no errors (no output, exit 0).

If no `jackson-annotations` jar exists anywhere on disk, fetch it via its Maven coordinate
`com.fasterxml.jackson.core:jackson-annotations` (any 2.x), e.g.:

    mvn dependency:get -Dartifact=com.fasterxml.jackson.core:jackson-annotations:2.17.2
    JACKSON_JAR="$HOME/.m2/repository/com/fasterxml/jackson/core/jackson-annotations/2.17.2/jackson-annotations-2.17.2.jar"

then run the `javac` command above.
